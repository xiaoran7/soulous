# Soulous 优化方案（v1 规划）

> 立项日期：2026-06-08。本文是「14 项优化」的架构与分阶段执行计划，逐项推进时按 Phase 勾选。
> 关联文档：`docs/architecture.md`（总体结构）、`docs/database.md`（迁移规约）、`docs/ai-review-rules.md`（AI 审核）。
>
> **✅ 落地状态（2026-06-09）**：14 项里 **12 项已实现并测试通过**（5 注册+邮箱码 / 10 课表分学期 / 7 金币 / 13·14 经验+动作解锁 / 3 打卡 / 9 惩罚 / 6 宠物市场 / 2 邮件提醒 / 12 我的页 / 4 落地页 / 8 共享自习室）。第 1 项(LLM 优化) 与第 11 项(复盘) 经用户确认**暂不改**。后端 238 测试 / 前端 41 测试全绿。本文以下为当时的规划内容，保留作设计依据。

## 0. 已确认的关键决策

| 议题 | 决策 |
|---|---|
| 共享自习室（项8） | **轻量在线状态**：只显示在线/在学成员 + 各自计时 + 人数，复用 SSE/轮询，**不引入 WebSocket / 音视频** |
| LLM 优化方向（项1） | **提示词/输出质量 + 稳定性兜底**两条，不做成本优化与新 AI 能力 |
| 首只宠物（项6） | 移除默认宠物；新用户 onboarding **免费从入门款里选一只**，其余款式金币解锁，市场全程可预览 |
| 未登录预览（项4） | **独立营销落地页**，点「开始使用」才进登录；真实页面仍全程鉴权（不开放只读后端接口） |
| 课表学期（项10） | 旧学期**保留**，课表固定显示本学期；成绩/考试拆成**独立分学期历史页**，消灭混合视图 |
| 复盘（项11） | **本期不改**，移出范围 |
| 货币命名（项7） | **金币** |
| 成长（项13/14） | 满级 **Lv30**，曲线 `expForNextLevel(L)=50+50·L+8·L²`；**每只宠物独立升级**，经验/等级不共享，只有出战宠物吃经验；金币账号级共享、经验宠物级独立 |

## 1. 依赖关系总览

```
            ┌─ 金币钱包(7) ───────────────┐
每日打卡(3) ─┤                            ├─→ 宠物市场+移除默认宠物(6) ─→ 我的页填充(12)
            └─ 经验/成长重做(13,14) ──────┘            │
                     │                                 └─→ 邮件提醒(2)
                  惩罚机制(9)
独立可并行：课表分学期(10)、注册页+邮箱(5)、LLM(1)、落地页(4)、轻量共享自习室(8)
```

核心结论：**宠物市场(6) 依赖金币(7) 与成长(13/14)，必须后做**；金币与成长是整套经济系统的地基，先落地。

## 2. 数据模型改动

> ⚠️ prod 为 `ddl-auto: validate`，每个新增字段/表必须配套 Flyway `V{n}__*.sql`，**h2 与 mysql 各一套**（当前 h2→V11、mysql→V12，版本号已分叉）。已应用迁移不可改内容（checksum）。

| 改动 | 内容 | 项 |
|---|---|---|
| `user_account` + `coin_balance` 列 | 账号级金币余额 | 7 |
| 新表 `coin_ledger` | 金币流水：来源类型、金额、余额快照、关联实体 | 7 |
| 新表 `pet_species` | 市场目录：slug、名称、稀有度、价格、sprite 路径、是否入门款、动作解锁配置 | 6 |
| `pet` 表改造 | `@OneToOne user` → `@ManyToOne user`；新增 `species_id` / `active`(出战标记) / `acquired_at`；level/exp/mood/satiety 保持**每行独立** | 6,13,14 |
| 新表 `daily_checkin` | user、date、streak、奖励快照 | 3 |
| 新表 `study_room` + `room_member` | 房间元信息 + 成员在线/计时状态 | 8 |
| `user_account.email` 采集 | 列已存在，注册流程补采集 | 5,2 |
| （可选）`user_notification_pref` | 按类型退订/频率偏好 | 2 |

**最高风险项：`pet` 一对一 → 多对一。** 影响面：
- `PetService.get()` 需改为「取出战宠物」，可能返回空（新用户未领养时）。
- 6 处 `UserService.ensurePet()` 删除/改造（不再自动建宠）。
- 所有 `markXxx` / `addExp` / `addFocusExp` 改为作用于「出战宠物」。
- 前端 `main.tsx` 的 `bootstrap()`、`ProfilePage`、`PetPage` 需处理 `pet === null` 空态。
- 老用户数据迁移：已有 Pet 赋默认 species + `active=true`。

## 3. 分阶段执行计划

### Phase 0 · 减法与独立项（低风险，可并行先做）

**[10] 课表分学期重构**
- 后端：`TimetableService` / `TimetableController` 课表查询固定「当前学期」，移除「全部学期」混合查询路径。
- 成绩/考试（`GradeEntry` / `ExamEntry` / `ExamGrade.tsx`）：保留全部历史，做成**独立分学期历史页**，学期切换只在成绩页内生效，不与课表混排。
- 前端 `TimetablePage`：删课表里的学期下拉，固定本学期。
- 验收：课表页无学期切换、只显示本学期；成绩页能按学期分别查看历史，互不串台。
- 注意：`GradeEntry`/`ExamGrade.tsx` 是近期新增（git status 未提交），改动前先确认现状，勿误删历史数据。

**[5] 注册页优化 + 补采集邮箱**
- 注册表单新增 `email` 输入（强引导填写，给项2 铺路）。
- UX：登录/注册改为带功能亮点的视觉版（现为纯表单）；密码强度做成实时可视条。
- 验收：注册可写入邮箱并回显于「我的」；密码强度实时反馈。

**[1] LLM 优化（提示词质量 + 稳定性兜底）**
- 提示词：重写 AI 审核（`AiReviewProcessor` + `ai-review-rules.md`）、AI 拆解（chat）、每日复盘三处 system prompt，加结构化输出约束 + few-shot，降低误判。
- 兜底：审核 LLM 不可用时的规则回退补全、流式中断重连、超时分级。
- 验收：构造 LLM 超时/异常场景，三处功能均有可用回退、不白屏。

### Phase 1 · 经济与成长地基

**[7] 金币系统**
- 新 `wallet` 模块：`CoinService`（加/扣/查，幂等流水）、`CoinLedger`、`coin_balance`。
- 赚币来源：完成任务、自律打卡、每日签到、专注完成。
- 花费：买宠物（Phase 2）；预留喂食/装扮。
- 验收：各来源加币写流水、余额一致；并发不超扣。

**[13/14] 经验曲线 + 动作解锁（每宠独立）**
- 曲线：满级 Lv30，`expForNextLevel(L)=50+50·L+8·L²`（替换现 `100+level*35`）。
- 倍率：现心情三档（1.2/1.0/0.8）叠加**连续打卡 streak 倍率（×1.0→×1.5 封顶）** 与饱腹感档位。
- **每宠独立**：经验只进出战宠物；切换宠物各自保留 level/exp/动作解锁；新宠从 Lv1 起。
- 动作解锁表（对齐 sprite 9 态 + `PetStage`）：

  | 等级 | 阶段 | 解锁动作 |
  |---|---|---|
  | Lv1 | EGG | idle、waiting |
  | Lv3 | CHILD | + waving、failed |
  | Lv8 | GROWING | + running(left/right) |
  | Lv15 | GROWING | + jumping |
  | Lv22 | ADULT | + review |
  | Lv30 | 满级 | 全 9 态解锁 |

- 验收：升级曲线生效；未解锁动作在 `PetSprite`/宠物页不可触发；多宠切换等级互不影响。

**[3] 每日打卡**
- 触发：每日首次打开 app 或完成当日首个任务 → +固定经验（出战宠物）+金币 +streak。
- `daily_checkin` 记录连续天数，驱动 streak 倍率。
- 验收：每日仅记一次；连续/断签 streak 正确。

**[9] 惩罚机制（温和衰减，不硬扣等级）**
- 断签：定时任务扫描，超 N 天未打卡 → 出战宠物 mood/satiety 衰减、转 SLEEPY/SAD（不掉级）。
- 任务逾期：计划任务到期未完成 → 小幅 mood 扣减。
- 保留现有审核驳回惩罚（mood-10），纳入统一惩罚视图。
- 验收：断签/逾期触发衰减；等级不被扣减。

### Phase 2 · 宠物市场 + 移除默认宠物（依赖 Phase 1）

**[6] 宠物市场**
- `pet_species` 目录 + `pet` 拥有实例改造（见 §2）。
- 素材：用 `hatch-soulous-pet` 技能预生成 3–5 款不同形象/稀有度上架。
- onboarding：注册后「领养你的第一只伙伴」，从入门款免费选一只（替代原默认宠物逻辑，删 `ensurePet`）。
- 市场页：未拥有款式可预览精灵 + 解锁动作 + 价格；金币足够才能买；可切换出战宠物。
- 老用户迁移：已有 Pet 赋默认 species + `active=true`。
- 验收：新用户无默认宠、可免费领一只；金币购买/切换出战正常；市场可预览未拥有款。

### Phase 3 · 邮件提醒

**[2] 邮件提醒（依赖 Phase 0 邮箱、Phase 1 打卡）**
- 定时调度（`@Scheduled`）：每日傍晚扫描「今日未打卡 / 有逾期任务」用户 → 走已有 `EmailNotificationSink` 发提醒。
- 配 SMTP：`soulous.notification.email.enabled=true` + `spring.mail.*`（复用用户级环境变量机制）。
- 可选：`user_notification_pref` 退订/频率偏好。
- 验收：未打卡用户傍晚收到邮件；已打卡不重复打扰；可退订。

### Phase 4 · 体验打磨

**[4] 营销落地页**
- 新建独立落地页（功能亮点 + 截图/精灵动效），未登录访问根路径看落地页，点「开始使用」进 `AuthScreen`。
- `main.tsx` 在 `if (!user)` 前加路由分流；真实页面仍全程鉴权，不开放只读后端。
- 验收：未登录看落地页；点 CTA 进登录；登录后正常进应用。

**[8] 轻量共享自习室**
- `FocusPage` 增「共享房间」模式：建房/进房，房内用 SSE/轮询展示在线成员 + 各自计时态 + 人数；不做聊天/音视频。
- 后端 `study_room` / `room_member`，复用 `NotificationSseService` 的 SSE 基建。
- 验收：多端进同一房可见彼此在线与计时；退出/超时正确下线。

**[12] 我的页填充**（依赖 Phase 1/2 数据）
- `ProfilePage` 现仅账号卡 + 宠物卡。新增：金币余额、连续打卡日历、经验/等级进度条、成就徽章、最近经验流水、宠物收藏墙。
- 验收：各数据真实联动；空态友好。

## 4. 推进顺序建议

1. Phase 0 三项并行（互不依赖，快速见效）。
2. Phase 1：先金币 → 再成长（13/14）→ 打卡(3) → 惩罚(9)。
3. Phase 2 宠物市场（地基齐全后）。
4. Phase 3 邮件、Phase 4 体验打磨收尾。

## 5. 未决小项

- 金币赚取/消费的**具体数值**（任务/签到/专注各给多少、入门款价格）——进入 Phase 1/2 时定。
- 断签惩罚的 **N 天阈值**与衰减幅度——进入 Phase 1 项9 时定。
